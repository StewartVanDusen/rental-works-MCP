  <action>
Create the `scripts/` directory and `scripts/fetch-swagger.ts` file. The script must:

1. Define the 12 sub-spec names as a constant array:
   `accountservices-v1`, `home-v1`, `warehouse-v1`, `settings-v1`, `pages-v1`, `reports-v1`, `utilities-v1`, `administrator-v1`, `mobile-v1`, `plugins-v1`, `integrations-v1`, `storefront-v1`

2. Derive the base URL from `process.env.RENTALWORKS_BASE_URL` with fallback to `"https://modernlighting.rentalworks.cloud"`.

3. For each sub-spec, fetch `${BASE_URL}/swagger/${specName}/swagger.json` (unauthenticated).

4. Parse the OpenAPI JSON, iterate `Object.entries(data.paths)`, and for each path iterate `Object.keys(methods)` to collect `{ method: method.toUpperCase(), path, spec: specName }` entries. Skip OpenAPI metadata keys like `parameters` by filtering to only HTTP method keys: get, post, put, delete, patch, options, head.

5. Write the output to `scripts/swagger-cache.json` with this structure:
   ```json
   {
     "generatedAt": "ISO timestamp",
     "baseUrl": "the base URL used",
     "subSpecs": [{ "name": "home-v1", "pathCount": 1845 }, ...],
     "totalPaths": 5000,
     "paths": [{ "method": "POST", "path": "/api/v1/order/browse", "spec": "home-v1" }, ...]
   }
   ```

6. Log progress to stdout: `Fetching {specName}...` and `  OK: {count} endpoints` per spec, plus a final summary line.

Use ESM imports (`import { writeFileSync } from "fs"`, `import { fileURLToPath } from "url"`, `import { dirname, join } from "path"`). Use `const __dirname = dirname(fileURLToPath(import.meta.url))` for script-relative paths. Top-level await is fine (tsx supports it).

Filter methods to only valid HTTP methods to avoid counting OpenAPI `parameters` objects as methods:
```typescript
const HTTP_METHODS = new Set(["get", "post", "put", "delete", "patch", "options", "head"]);
// In the loop:
for (const method of Object.keys(methods)) {
  if (!HTTP_METHODS.has(method.toLowerCase())) continue;
  // ...
}
```
